## Research: i18n Approach Selection

### Approaches Evaluated

| Approach | Viable? | Rationale |
|---|---|---|
| **gettext + Babel + Jinja2 i18n ext** | **Yes (recommended)** | Battle-tested (30+ years), native Jinja2 support via `jinja2.ext.i18n`, used by Django/Flask-Admin/starlette-admin. Zero runtime overhead (dict lookup). Standard `.po`/`.mo` format supported by all translation platforms. |
| Mozilla Project Fluent (`fluent.runtime`) | **No** | No Jinja2 integration exists. Last release Sep 2023 (unmaintained). 237 GitHub stars, unresolved PRs. Would require building custom Jinja2 extension and extraction tool from scratch. |
| Browser-based translation (Google Translate widget) | **No** | Unacceptable quality for admin UI short strings. Privacy/GDPR concern (sends page content to Google). No RTL handling. Inappropriate to embed third-party widget in a library. |
| **Pluggable TranslationProvider protocol** | **Yes (layer on top)** | Thin protocol class with gettext default. Matches existing HyperAdmin patterns (`auth_backend`, `storage` are already pluggable). Future-proof without over-engineering. |

### Competitor Comparison

| Framework | Backend | Built-in Locales | Negotiation |
|---|---|---|---|
| **Django Admin** | gettext + `makemessages` | ~90 languages | Accept-Language, settings, URL prefix, middleware |
| **Flask-Admin** | Flask-Babel (gettext + Babel) | English only (community provides others) | Accept-Language via Babel locale selector |
| **starlette-admin** | Babel + gettext | en, fr, de, ru, tr | `I18nConfig(default_locale=..., supported_locales=[...])` |
| **sqladmin** | None | English only | N/A |

### Recommended Architecture

```
src/hyperadmin/i18n/
  __init__.py          # TranslationProvider protocol + GettextProvider default
  config.py            # I18nConfig(default_locale, supported_locales)
  middleware.py         # LocaleMiddleware: cookie -> settings -> Accept-Language -> "en"
  babel.cfg            # Babel extraction config for Jinja2 templates

src/hyperadmin/locale/
  en/LC_MESSAGES/hyperadmin.po
  es/LC_MESSAGES/hyperadmin.po
  fr/LC_MESSAGES/hyperadmin.po
  de/LC_MESSAGES/hyperadmin.po
  zh_CN/LC_MESSAGES/hyperadmin.po
  ja/LC_MESSAGES/hyperadmin.po
  uk/LC_MESSAGES/hyperadmin.po
```

### Locale Negotiation Chain (priority order)

1. Cookie (user-selected via language picker)
2. `HyperAdminSettings.locale` (constructor default)
3. `Accept-Language` header (browser preference)
4. Fallback to `"en"`

### New Dependencies

- `babel>=2.12` — runtime dep (locale negotiation, number/date formatting, message compilation)
- Python `gettext` — stdlib (no new dep)
- Jinja2 `ext.i18n` — already bundled with `jinja2>=3.1.6`

### RTL Strategy

- Migrate 16 physical CSS properties (`margin-left`, `padding-right`, etc.) to CSS logical properties (`margin-inline-start`, `padding-inline-end`)
- `[dir="rtl"]` selectors for edge cases (SVG icon flip, custom positioning)
- `<html lang="{{ locale }}" dir="{{ 'rtl' if locale in rtl_locales else 'ltr' }}">`
- RTL locales (ar, he) deferred to v0.4.2+ unless user demand

### Translation Scope

~50 translatable strings across ~35 templates. Categories:
- Navigation/chrome: "Menu", "Admin", "Logout", "Toggle dark mode", etc.
- CRUD operations: "Create", "Update", "Delete", "Save", "Cancel", etc.
- Search/filter: "Search...", "Filters", "Clear all filters", etc.
- Pagination: "Showing X to Y of Z results", "Previous", "Next", etc.
- Auth: "Sign in", "Username", "Password", etc.
