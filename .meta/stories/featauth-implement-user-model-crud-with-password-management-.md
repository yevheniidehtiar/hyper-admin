## Context

A `User` model already exists in `examples/rbac_app/models.py` with fields: `id`, `username`, `email`, `first_name`, `last_name`, `is_active`, `is_superuser`, `created_at`, `updated_at`, `avatar`, `personal_key`. However:

1. It has no `password_hash` field (blocked on #119 which adds it)
2. There is no `UserAdmin` class registered to expose User CRUD in the admin interface

This issue wires up the User model to the admin interface with safe password handling.

**Depends on:** #119 (adds `password_hash` field and `argon2-cffi`)

## Acceptance Criteria

- [ ] `UserAdmin` class is defined in `examples/rbac_app/admin.py` and registered with the admin
- [ ] `column_list` includes: `id`, `username`, `email`, `is_active`, `is_superuser`, `created_at`
- [ ] `column_searchable_list` includes: `username`, `email`
- [ ] `column_sortable_list` includes: `id`, `username`, `created_at`
- [ ] `column_filters` includes: `is_active`, `is_superuser`
- [ ] Create form includes `username`, `email`, `first_name`, `last_name`, `is_active`, `password` (plain text input), `password_confirm` fields
- [ ] On create/update, plain-text `password` is hashed via `auth.backend.hash_password()` before persisting — the raw value is never stored
- [ ] `password_hash` is excluded from `form_columns` and `column_list` (never shown in UI)
- [ ] Update form shows a dedicated "Change Password" section (optional — leave blank to keep existing hash)

## Implementation Notes

**Modified files:**
- `examples/rbac_app/admin.py` — add `UserAdmin(ModelView, model=User)` with the column/form config above
- `examples/rbac_app/models.py` — ensure `password_hash` has `exclude=True` so it is never serialized to the frontend

**Password field handling** — override the create/update hooks in `UserAdmin` (if `ModelView` supports `on_create` / `on_update` hooks; otherwise patch in `views/dynamic.py`):
```python
async def on_model_change(self, data: dict, model: User, is_created: bool) -> None:
    if plain := data.pop("password", None):
        data["password_hash"] = hash_password(plain)
```

**Template:** No custom template needed — standard list/create/update views are generated by `DynamicModelView` (`src/hyperadmin/views/dynamic.py`).

## Testing Requirements

**Unit tests** (`tests/unit/test_user_admin.py`):
- Creating a user via the admin stores a hash, not the plain text password
- Leaving password blank on update keeps the existing hash

**E2E tests** (`tests/e2e/auth/test_user_crud.py`):
- User list renders with `data-testid="list-table"` and shows correct columns
- Create user form accessible via `data-testid="create-link"`
- Form fields present via `page.get_by_label("Username")`, `page.get_by_label("Email")`, etc.
- Submitting create form with valid data creates the user and shows it in the list
- Submitting with duplicate username shows `data-testid="username-errors"`
