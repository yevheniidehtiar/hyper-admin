## Context

All admin routes generated by `create_admin_router()` (`src/hyperadmin/routing.py`) are currently publicly accessible. This issue gates every admin route behind authentication and wires per-model permission checks into `DynamicModelView` (`src/hyperadmin/views/dynamic.py`).

**Depends on:** #119 (session/auth backend), #115 (Permission model), #116 (User model)

## Acceptance Criteria

**Global auth guard:**
- [ ] Every route mounted under the admin prefix requires an active session (logged-in user)
- [ ] Unauthenticated requests receive a 302 redirect to `/admin/login` (not a 401 JSON response ‚Äî the admin is HTML-first)
- [ ] The guard is implemented as a FastAPI dependency `require_authenticated_user(request: Request) -> User` in `src/hyperadmin/auth/middleware.py`
- [ ] `create_admin_router()` in `src/hyperadmin/routing.py` accepts the dependency and applies it via `dependencies=[Depends(require_authenticated_user)]`

**Per-model permission checks:**
- [ ] `AdminOptions` (`src/hyperadmin/core/options.py`) already has `can_create`, `can_edit`, `can_delete` boolean flags ‚Äî these must now be evaluated against the current user's permissions in addition to the static flags
- [ ] Superusers (`User.is_superuser == True`) bypass all permission checks
- [ ] Non-superuser attempting a disallowed action receives HTTP 403 with a user-facing error message rendered in the admin template (not a raw JSON response)
- [ ] The permission codename convention: `{action}_{model_name_lower}` (e.g., `create_user`, `edit_group`, `delete_permission`)

**Logout:**
- [ ] `GET /admin/logout` is implemented and clears the session (may be implemented in #119 but verified here end-to-end with permission context)

## Implementation Notes

**Modified files:**
- `src/hyperadmin/auth/middleware.py` ‚Äî add `require_authenticated_user` dependency (reads `request.session["user_id"]`, queries DB for `User`)
- `src/hyperadmin/routing.py` ‚Äî thread the auth dependency into `create_admin_router()`; make it opt-out via `auth_enabled: bool = True` parameter so the example app without auth still works
- `src/hyperadmin/views/dynamic.py` ‚Äî inject current user context into view handlers; check `AdminOptions` flags + user permissions before executing create/update/delete operations
- `src/hyperadmin/core/options.py` ‚Äî no structural changes needed; flags are already present

**Dependency direction (CONSTITUTION.md):** `views/dynamic.py` may import `auth/middleware.py`; `auth/` must NOT import from `views/`.

## Testing Requirements

**Unit tests** (`tests/unit/test_authorization.py`):
- `require_authenticated_user` raises `HTTPException(302)` when session has no `user_id`
- `require_authenticated_user` returns `User` when session contains valid `user_id`
- Superuser passes all permission checks
- Non-superuser with matching permission codename passes the check
- Non-superuser without matching permission codename raises `HTTPException(403)`

**E2E tests** (`tests/e2e/auth/test_authorization.py`):
- Unauthenticated `GET /admin/` ‚Üí redirected to `/admin/login`
- Authenticated superuser can access list, create, edit, delete for all models
- Authenticated non-superuser without `delete_user` permission: delete button absent or returns 403
- Use `page.get_by_role("button", name="Delete")` ‚Äî not CSS selectors
